Manejo de Excepciones en C++

El manejo de excepciones en C++ permite gestionar errores y condiciones excepcionales durante la ejecución del programa de una manera estructurada y segura. Utiliza el mecanismo de try, catch, y throw para detectar y manejar errores.

Conceptos Básicos

El manejo de excepciones se basa en tres palabras clave principales:

  • try: Bloque de código en el que se puede producir una excepción.
  • catch: Bloque de código que maneja la excepción lanzada por el bloque try.
  • throw: Palabra clave para lanzar una excepción.

Ejemplo Básico

El siguiente ejemplo muestra cómo usar try, catch, y throw para manejar una excepción:

                
                    #include <iostream>
            
                    void dividir(int a, int b) {
                        if (b == 0) {
                            throw std::runtime_error("División por cero"); // Lanzar excepción
                        }
                        std::cout << "Resultado: " << a / b << std::endl;
                    }
            
                    int main() {
                        try {
                            dividir(10, 0); // Intentar ejecutar código que puede lanzar excepción
                        } catch (const std::runtime_error &e) {
                            std::cerr << "Error: " << e.what() << std::endl; // Manejar excepción
                        }
                        return 0;
                    }

En este ejemplo, la función dividir lanza una excepción si el divisor es cero. El bloque catch en main captura y maneja la excepción.

Tipos de Excepciones

En C++, las excepciones pueden ser de cualquier tipo, pero las más comunes son:

  • std::exception: La clase base para todas las excepciones estándar.
  • std::runtime_error: Para errores que ocurren durante la ejecución del programa.
  • std::logic_error: Para errores en la lógica del programa.

Propagación de Excepciones

Las excepciones se propagan a través de la pila de llamadas, lo que significa que si una función no maneja una excepción, la excepción se pasará a la función que la llamó, y así sucesivamente, hasta que sea manejada o hasta que el programa termine.

Ejemplo de Propagación de Excepciones

                
                    #include <iostream>
            
                    void funcionB() {
                        throw std::runtime_error("Error en funcionB");
                    }
            
                    void funcionA() {
                        funcionB(); // Propaga la excepción a la función que la llamó
                    }
            
                    int main() {
                        try {
                            funcionA(); // Intentar ejecutar código que puede lanzar excepción
                        } catch (const std::runtime_error &e) {
                            std::cerr << "Error: " << e.what() << std::endl; // Manejar excepción
                        }
                        return 0;
                    }

Excepciones Personalizadas

También puedes definir tus propias clases de excepciones derivadas de std::exception para manejar errores específicos de tu aplicación.

                
                    #include <iostream>>
                    #include <stdexcept>
            
                    class MiExcepcion : public std::exception {
                    public:
                        const char* what() const noexcept override {
                            return "Mi excepción personalizada";
                        }
                    };
            
                    void funcion() {
                        throw MiExcepcion(); // Lanzar excepción personalizada
                    }
            
                    int main() {
                        try {
                            funcion(); // Intentar ejecutar código que puede lanzar excepción
                        } catch (const MiExcepcion &e) {
                            std::cerr << "Error: " << e.what() << std::endl; // Manejar excepción personalizada
                        }
                        return 0;
                    }

En este ejemplo, MiExcepcion es una clase personalizada que hereda de std::exception. El método what proporciona un mensaje de error específico.
